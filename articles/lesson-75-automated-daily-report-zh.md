# 第 75 讲：用 OpenClaw 做自动化日报：定时采集、分析、生成和发送

自动化日报不是每天复制一堆数字。真正有用的日报应该稳定采集、自动计算、突出异常、给出解释，并在发送前有必要的确认。

## 用到的 Skill

优先到 [skills.lc](https://skills.lc/) 搜索 `automation`、`report`、`schedule`、`dashboard`。当前环境可对应使用定时任务能力，使用 `browser` 采集网页数据，使用 `spreadsheets` 汇总指标，使用 `documents` 或 `email` 相关 Skill 生成和发送日报。

## 执行链路

先定义日报模板：核心指标、来源、更新时间、计算口径、异常阈值、接收人。

再定义采集方式：API、网页、文件、数据库或手工表格。每个来源都要有失败处理。

然后自动分析：同比、环比、目标差距、异常点和可能原因。日报不需要长，但要把变化说清楚。

最后发送或待确认发送。高风险日报可以先生成草稿，让负责人审核后再发。

## 人工确认点

涉及财务、客户、故障、事故和公开披露的信息，不能无确认自动发送。

## 实操练习

让 OpenClaw 设计一个“产品运营日报”：每天采集访问、注册、转化和错误数，生成摘要并发送邮件草稿。

自动化日报的价值，是让团队每天看到同一套可信事实。
