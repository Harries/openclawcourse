# 第 58 讲：用 OpenClaw 写长文档：方案书、报告、产品文档和排版检查

长文档最怕两件事：前后口径不一致，以及写完没人愿意读。OpenClaw 写长文档的优势，不是一次生成几万字，而是能把“资料整理、章节规划、逐段生成、交叉检查、排版验收”拆成稳定步骤。

## 用到的 Skill

优先到 [skills.lc](https://skills.lc/) 搜索 `document`、`docx`、`report`、`writing`。当前环境可对应使用 `documents:documents` 创建和渲染 `.docx`；用 `browser` 查资料；用 `summarize` 摘要外部材料；用 `spreadsheets` 放入表格和图表；用 `imagegen` 生成示意图。

## 执行链路

先收集素材：原始需求、访谈记录、数据表、旧文档、参考链接。OpenClaw 不应该直接“编一篇”，而是先生成素材清单和缺口清单。

再做文档架构：标题层级、每章目标、关键证据、预计字数。长文档必须先有结构锁定，否则后面越写越散。

然后分章写作。每写完一章，就让 OpenClaw 做三类检查：是否回答本章问题，是否和前文重复，是否引用了未经确认的事实。

最后用 Documents Skill 生成文档并渲染检查。页眉页脚、目录、表格跨页、图片说明、标题层级都要看。

## 人工确认点

合同报价、交付承诺、客户名称、产品参数、法律表述和统计结论必须人工确认。OpenClaw 可以提示风险，不能替你承担责任。

## 实操练习

写一份《企业引入 OpenClaw 的试点方案》，输出 Markdown 草稿和 `.docx` 成稿，并附一页“待客户确认问题”。

长文档不是一口气写出来的，而是一轮轮收敛出来的。
