# Lesson 66: Web Research with OpenClaw

Web research is not copying search results. Reliable research defines the question, selects sources, cross-checks facts, keeps citations, and separates evidence from conclusions.

## Skills Used

Search [skills.lc](https://skills.lc/) for `browser`, `web research`, `scrape`, or `summary`. In this environment, use `browser:control-in-app-browser` for page inspection, `summarize` for URLs, `documents` for reports, and `spreadsheets` for source matrices.

## Workflow

Define the research question first. “Is this tool suitable for a small team?” is better than “look this up.”

Plan sources: official site, docs, GitHub, pricing page, changelog, user reviews, and third-party analysis.

Extract facts page by page: claim, link, date, screenshot need, and uncertainty. Then write a report with conclusions first and evidence after.

## Human Review

Web information changes. Pricing, policies, features, company status, and legal terms must be checked against current pages.

## Exercise

Research three agent tools and produce a feature matrix, source links, and a one-page purchase recommendation.

The baseline is simple: every important judgment should point back to a source.
