# Lesson 57: Creating a Complete PPT with OpenClaw

A usable deck is not a pile of generated bullets. It needs an audience, a story, a slide plan, visual checks, source checks, and a final file that can actually be presented.

## Skills Used

Search [skills.lc](https://skills.lc/) first with `presentation`, `slides`, `ppt`, or `deck`. In this environment, the closest skills are `presentations:Presentations` for PowerPoint files, `imagegen` for visual assets, `browser` for research, `spreadsheets` for charts, and `documents` for speaker notes.

## Workflow

Start with a briefing: topic, audience, time limit, slide count, tone, required claims, and forbidden claims. OpenClaw should turn that into a slide outline before creating a file.

Then build a slide plan. Each slide should have a title, one key message, a visual idea, and speaker notes.

Generate the `.pptx` with the presentation skill, then render and inspect it. Check overflowing titles, unclear charts, inconsistent colors, missing page numbers, and unsupported claims.

## Human Review

Customer data, financial numbers, pricing, legal claims, and delivery promises must be confirmed by a human.

## Exercise

Ask OpenClaw to create a 10-slide deck called “Why Businesses Need Agent Workflows,” including a comparison slide for manual workflows versus agent workflows.

The point is not to let AI decorate slides. The point is to turn research, structure, visuals, and review into a repeatable production workflow.
