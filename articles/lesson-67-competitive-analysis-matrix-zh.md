# 第 67 讲：用 OpenClaw 做竞品分析：网页、表格、截图和对比矩阵

竞品分析不是罗列对手功能，而是回答：用户为什么会选它，为什么会离开它，我们应该学什么、避开什么。

## 用到的 Skill

优先到 [skills.lc](https://skills.lc/) 搜索 `competitive analysis`、`browser`、`screenshot`、`spreadsheet`。当前环境可对应使用 `browser` 访问竞品页面和截图，使用 `spreadsheets` 制作对比矩阵，使用 `presentations` 做汇报页，使用 `documents` 写结论。

## 执行链路

第一步，确定比较维度：目标用户、核心场景、价格、关键功能、集成能力、部署方式、安全合规、上手成本。

第二步，采集证据。官网文案、文档、价格页、截图、公开案例和用户评价都要保存来源。

第三步，填对比矩阵。每个格子尽量写可验证事实，不写“感觉很好”。

第四步，输出判断：哪些是行业标配，哪些是差异化机会，哪些功能不值得跟。

## 人工确认点

不要攻击竞品；不要使用未授权截图做公开宣传；不要把用户评论当成统计结论。

## 实操练习

选择三个知识库产品，让 OpenClaw 输出一张对比矩阵、一页机会分析和一组截图证据清单。

好的竞品分析不是证明自己更好，而是帮助团队少走弯路。
