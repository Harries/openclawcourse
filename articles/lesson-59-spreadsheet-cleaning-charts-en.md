# Lesson 59: Cleaning Spreadsheets and Creating Charts with OpenClaw

Spreadsheet work is risky because small data problems can lead to wrong conclusions. Before charting anything, OpenClaw should make the data trustworthy.

## Skills Used

Search [skills.lc](https://skills.lc/) for `spreadsheet`, `excel`, `csv`, or `chart`. In this environment, use `spreadsheets:Spreadsheets` for Excel and CSV work, `documents` for written analysis, `presentations` for reporting slides, and `browser` for external definitions.

## Workflow

Start with a data health check: sheets, rows, columns, headers, missing values, duplicates, date formats, currency formats, and suspicious outliers.

Define cleaning rules before changing data. For example, normalize dates to `YYYY-MM-DD`, convert currency to one unit, remove exact duplicate rows, and keep repeated purchases when they are legitimate events.

Then create pivot tables and charts that match the question: line charts for trends, bar charts for rankings, stacked bars for composition, and scatter plots for relationships.

## Checks

Compare row counts before and after cleaning. Confirm totals did not change unexpectedly. Make sure chart titles are specific and conclusions do not go beyond the data.

## Exercise

Give OpenClaw a sales workbook and ask for a cleaned workbook, three charts, and a one-page business summary.

Spreadsheet automation turns messy inputs into facts people can discuss.
